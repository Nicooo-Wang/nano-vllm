# 课程开发角色定义与质量标准

本文件定义 nano-vllm 课程开发的三个协作角色，以及所有角色共用的**六维质量 checklist**。每节课（L01–L08）的开发都以本文件为准派角色、按 checklist 验收。

## 三个角色

### 1. 框架开发工程师
- 产出每节课的 `TUTORIAL.md`、`lab.py`、`solution/`（参考答案 + 验证脚本）。
- **交付前**按下方六维 checklist 自检。
- 必须真跑通并自证成功判据（如 `All checks passed ✓`）。
- 根据审核 + 学生反馈持续修改。

### 2. 课程架构审核工程师（Reviewer）
- 对照 nano-vllm 源码 + 该课 spec，**逐维**审计全部产物。
- 输出优先级化问题清单：**阻塞 / 重要 / 次要**，每条带 `file:line` + 改法。
- 每轮迭代后重新审核，直到无阻塞/重要项。

### 3. 学生（Student）
- **盲测**：只许看 `TUTORIAL.md` + `lab.py`（**不许**看 `solution/`）。
- persona：算子开发者（系统/内核/推理优化背景），懂 LLM 推理原理，会读 Python 但不精通，**不熟 vLLM / nano-vllm**。
- 从学习者视角报问题：能不能照着讲义一步步做完？哪里卡住 / 误解？

## 六维质量 checklist（每个角色都查，视角不同）

| 维度 | 查什么 | 红旗（命中即问题） |
|---|---|---|
| ① 语言与受众一致 | 注释、docstring、讲义正文、图里文字是否用受众语言（中文）？运行时输出语言是否为**有意决策**（而非遗漏）？ | 中文受众的 `lab.py` 满篇英文注释；输出语言像随手没改 |
| ② 图示可读性 | 有无中英文混排的 ASCII 框图（双宽度字符会错位）？结构图用 **mermaid** 或纯英文 ASCII；表格用 **markdown** | `│ 中文 │` 混在英文框图里；用空格手工对齐的中英混排表 |
| ③ 讲义语气 | 讲义是"预先验证、直接可跑"的纲领性步骤 | "为什么不直接 X / 这里有个坑 / 踩过的坑 / 会失败" 等开发过程叙述 |
| ④ 讲义↔lab 对应与验证 | 讲义 step-by-step、与 `lab.py` **1:1** 对应；每个 Task 有明确输入/输出；**显式**写出"如何验证做对"（run `X` → 期望 `All checks passed ✓`） | 讲义没说怎么验证；学生要做的事在讲义里找不到对应 |
| ⑤ 正确性 | 每个 `file:line` 锚点对源码准确；代码真能跑；验证真能过 | 锚点行号错；引用不存在的东西；验证形同虚设 |
| ⑥ 教学法 | 目标清晰、知识递进合理、tut:lab 比例、altitude 符合 spec | 概念跳跃；为留空而留空；难度/粒度偏离 spec |

## 一节课的开发循环
1. 工程师按 spec 产出 TUTORIAL + lab + solution，自检六维 + 自证跑通。
2. 审核工程师按六维审计，出问题清单。
3. 学生盲测，报学习者视角问题。
4. 工程师据审核 + 学生反馈修改。
5. 再审 / 再试，直到无阻塞 / 重要项。

## 范围约定
- **运行时输出语言**是每节课的有意决策（默认与受众一致）。L01 经用户确认：`lab.py` 注释 / docstring 用中文，**运行时打印输出保持英文**。
- 讲义正文语言：与受众一致（中文）。
- 代码标识符、文件路径、`file:line` 锚点、shell 命令——保留原文，不翻译。
